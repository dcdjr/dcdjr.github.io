// 01 — About

I'm at URI studying CS with a Data Science minor, in the Honors College as a Schilling Scholar. Most of my time goes into projects that require understanding something at a low level — building real tools rather than just using them.

Right now I'm building Hivecore, a terminal-first local AI assistant, and tcom, an 8-bit computer emulator from scratch. I also compete in security competitions and run a self-hosted home server for the same reason I do most of this: because figuring out how something works means actually building it.

Long term I want to work somewhere at the intersection of security and real systems — the kind of work that requires knowing what's happening at the hardware and OS layer.

// 02 — Projects

Hivecore
Python Local-First CLI Systems Design

A terminal-first personal assistant built for local, private operation. Persistent memory, a managed home directory, run history, and explicit tool connectors — no cloud dependencies. The premise: your assistant should live on your machine, not someone else's server.

↗ GitHub
tcom
C Emulation Computer Architecture

An 8-bit computer emulator being built from scratch — CPU, memory, instruction set, all implemented manually. The goal is simple: understand how machines work by building one.

↗ GitHub
HTTP Server
C++ POSIX Sockets Concurrency Networking

A lightweight HTTP/1.1 server over raw POSIX sockets. Request parsing, static file serving, MIME detection, thread-per-connection concurrency, timeouts, request size limits, and directory traversal protection — no frameworks.

↗ GitHub
Unix Shell
C POSIX Process Control

A minimal Unix shell in C. Command parsing, external execution via fork/execvp, built-in commands, and signal handling that correctly targets child processes on interrupt.

↗ GitHub
TCP Port Scanner
C Winsock2 pthreads Security

A multithreaded port scanner in C using Winsock2 and pthreads. Queue-based worker architecture with banner grabbing, service detection, configurable timeouts, and per-scan timing stats.

↗ GitHub
Home Server
Ubuntu Server Linux Tailscale Self-Hosted

Self-hosted Ubuntu server with ~3TB of storage. OS install, service configuration, and ongoing administration — all done personally. Remote access via Tailscale. Mostly exists because running your own infrastructure teaches things that cloud consoles don't.
